# Project Specification Directory

This directory contains internal project governance and specification management tools used during development.

## Contents

- **`memory/constitution.md`** - Core architectural principles and non-negotiable rules for the Observer project
- **`templates/`** - Specification templates for feature planning and task management
- **`scripts/`** - Automation scripts for specification workflow

## Purpose

These files document the project's architectural decisions, development principles, and planning processes. They serve as:

1. **Architecture Documentation** - Core design principles and boundaries
2. **Development Guidelines** - Non-negotiable rules for code changes
3. **AI Agent Context** - Information for development workflow automation tools

## For Contributors

While these files document our internal development process, contributors should focus on:

- [CONTRIBUTING.md](../CONTRIBUTING.md) - How to contribute
- [README.md](../README.md) - Project overview and architecture
- [SECURITY.md](../SECURITY.md) - Security policies
- Code documentation in source files

The principles documented here inform our codebase design, but you don't need to modify these files to contribute features or fixes.
